# ARP位址解析協定
- 所謂位址解析就是主機在發送訊框前將目標IP位址轉換成目標MAC位址的過程
- ARP協定的基本功能就是通過目標設備的IP位址，查詢目標設備的MAC位址，以保證通信的順利進行
- 在每台安裝有TCP/IP協定的電腦裡都有一個ARP緩存表，表裡的IP位址與MAC位址是一一對應的
- 例子：我們以主機A（192.168.1.5）向主機B（192.168.1.1）
    1. 當發送資料時，主機A會在自己的ARP緩存表中尋找是否有目標IP位址
        - 如果找到了，也就知道了目標MAC位址，直接把目標MAC位址寫入訊框裡面發送就可以了
       
    2. 如果在ARP緩存表中沒有找到相對應的IP位址，主機A就會在網路上發送一個廣播，目標MAC位址是FF.FF.FF.FF.FF.FF，，這表示向同一網段內的所有主機發出這樣的詢問：“192.168.1.1的MAC位址是什麼？”
    3. 網路上其他主機並不回應ARP詢問，只有主機B接收到這個幀時，才向主機A做出這樣的回應：“192.168.1.1的MAC位址是00-aa-00-62-c6-09”。
    4. 主機A就知道了主機B的MAC位址，它就可以向主機B發送資訊了。同時它還更新了自己的ARP緩存表
    - ARP緩存表採用了老化機制，在一段時間內如果表中的某一行沒有使用，就會被刪除，這樣可以大大減少ARP緩存表的長度，加快查詢速度。

# ARP欺騙
- ARP spoofing
- 是針對以太網路位址解析協議（ARP）的一種攻擊技術
- 此種攻擊可讓攻擊者取得區域網路上的資料封包甚至可篡改封包，且可讓網路上特定電腦或所有電腦無法正常連線
- 運作原理
    - 由攻擊者發送假的ARP封包到網路上，尤其是送到閘道器上
    - 其目的是要讓送至特定的IP位址的流量被錯誤送到攻擊者所取代的地方
    - 攻擊者可將這些流量另行轉送到真正的閘道或是篡改後再轉送
    - 攻擊者亦可將ARP封包導到不存在的MAC位址以達到阻斷服務攻擊的效果
- 防治方法
    - 最理想的防制方法是網路內的每台電腦的ARP一律改用靜態的方式
        - 不過這在大型的網路是不可行的，因為需要經常更新每台電腦的ARP表。

# IP Spoofing
- 該方法使用偽造的來源位址來傳送 IP 封包
- 此類封包可能會採用受信任來源的 IP 位址來嘗試規避防火牆，這將使防火牆誤以為來自駭客的封包確實是來自受信任來源的
- IP 詐騙也可以僅用於隱藏攻擊的真實來源
- IP 詐騙是入侵者使用其他電腦的 IP 位址來取得資訊或存取權限的程序。由於入侵者偽裝成他人出現，因此如果傳送回覆，該回覆將發往入侵者所偽造的位址，而非其入侵者的位址。

# 交換器的廣播風暴
- 假設有一台伺服器發送廣播封包出來，並讓交換器A收到，交換器A再將封包交給交換器B，如果交換器配置不良，就會讓交換器B再發送封包給A，最後行程回圈，整個頻寬充斥著廣播封包
